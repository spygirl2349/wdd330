class Turtle {

constructor(name) {
this.name = name;
this.weapon = 'hands';
}

sayHi() {
return `Hi dude, my name is ${this.name}`;
}

attack(){
return `Feel the power of my ${this.weapon}!`;
} }

  • Use static to create a static method, it is not available to instances of the class
  • Create a new instance of a class:
    • const leo = new Turtle('Leonardo');
  • When using classes and prototypes to create instances, the following should be considered:
    • Create a class declaration that deals with any initialization, shared properties and methods
    • Use the prototype to augment extra methods and properties (this includes all instances, even those that have already been created)
    • An instance can be augmented using assignment of properties or methods (invididual to that instance) to that object
    • Be careful when overwriting the prototype completely -- the constuctor class needs to be reset
  • Example
    • const mike = new Turtle('Michelangelo');
    • mike.eat() // 'Mmm, this Pizza tastes great!'
    • mike.weapon = 'Nunchakus';
    • mike.attack(); // 'Feel the power of my Nunchakus!'
  • Add an underscore before a variable/method to make it private: let _color = blue;
  • Use getters and setters
  • Creat an instance of an object using the Object.create() method

Libraries

  • Libraries can help you reduce the code you write.
  • JQuery, DOM Manipulation, Underscore, and Lodash are all examples of libaries
  • Pros of libraries:
    • Popular libraries can make your code as bullet-proof as possible in many browsers
  • Cons:
    • The code for the library must be included in your code (can cause performance issues)
    • They can fail to implement the functionality in the way you intended
  • Browsers will cache libraries after the first download

Modular JavaScript

  • A module is way to reuse and organize your code which improves code mantainablility
  • Modules allow a public API to be exposed, but keep the implementation hidden away inside the module
  • Keeping code modular makes it more loosley coupled and interchangable
  • A few important points about modules:
    • All code in modules is always in strict mode and there's no way to opt out of this
    • a module has it's own global scope, so any variables in a top-level of module can only be accessed within that module
    • The value of this in the top level of a module is undefined, rather than the global object
    • You can't use HTML-style comments in modules
  • How to Export/Import Modules:
    • Export a very simple Pi Module:
      • export const PI = 3.1415926;
    • Import your Pi module into you main JS file:
      • import { PI } from './pi.js';
    • Use the * symbol to import everything from the specified module file
    • Default exports can be imported with having to be expicitly named
      • export default myClass
